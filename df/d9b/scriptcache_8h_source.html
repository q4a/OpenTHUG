28 // The total size of the CScriptCacheEntry pool.
29 // This is the max number of decompressed scripts that can exist at once.
30 // The code will assert if this number of scripts is exceeded.
31 // This may be set to be quite large since the CScriptCacheEntry is quite small, 24 bytes each.
32 // It needs to be fairly large to allow for spikes in the number of scripts executing (see TT1417)
33 #define MAX_DECOMPRESSED_SCRIPTS 300
34 
35 // The code will attempt to keep the total number of decompressed scripts to no more than this value
36 // by deleting unused scripts.
37 // This allows us to control the average number of decompressed scripts whilst still allowing spikes
38 // up to MAX_DECOMPRESSED_SCRIPTS without the code asserting.
39 // Allowing spikes is necessary because sometimes a lot of scripts may exist which don't use up much
40 // script heap altogether.
41 
42 // The larger IDEAL_MAX_DECOMPRESSED_SCRIPTS is, the less often the code will have to
43 // decompress scripts during gameplay, hence less cpu time will be used.
44 // However, the bigger it is, the more script heap will be used up storing decompressed scripts.
45 #define IDEAL_MAX_DECOMPRESSED_SCRIPTS 100
