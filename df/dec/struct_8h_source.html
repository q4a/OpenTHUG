239  // This will copy the contents of this structure into p_dest, but in such a way that
240  // p_dest will contain no unnamed-structure references, they will all get expanded.
241  // This was added because sometimes one wants to step through each of the components of
242  // a structure using GetNextComponent, but GetNextComponent itself cannot recurse into
243  // unnamed structures, because when it reaches the end of one it won't know how to get back
244  // to the parent.
245  // So instead just create a new structure, copy the other structure into it using ExpandInto,
246  // then step through that using GetNextComponent, then delete the new structure once finished.
247  // It's a bit memory intensive, but saves having to write special code to resolve unnamed
248  // structures & recursing whenever one wants to scan through the components.
249  // Could be handy for other things later too.
250  // recursionCount is just to catch infinite recursion, so no need to pass it a value.
