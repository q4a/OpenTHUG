24 // The Sector Story:
25 //
26 // To the designer, A "Sector" (previously a "world sector" is an
27 // object in 3D Studio Max. Typically this might be a tree, a wall
28 // a section of a building, a parked car or any other static part of the world
29 //
30 // The designer wants to be able to:
31 // - load the sector as part of the scene, see it and collide with it
32 // - reference the sector by name
33 // - Turn it on and off (both collision and display)
34 // - make it visible and invisible in particular viewports for LOD reasons
35 // - change its color
36 // - change various attributes, like UV wibble speed.
37 //
38 // To the engine programmer, the sector will be split into various bits of data
39 // for optimization and platform specific reasons. The geometry might be in different chunks
40 // and the collision data might be seperate from the rendering data.
41 // The act of "turning off" a sector will be very different from platform to platform
42 //
43 // To get round this problem, we have a CSector class which acts as a facade
44 // to the actual machine specific sector data.
45 //
46 // When the engine loads a scene, it will create a list of CSectors, one for each
47 // sector in the scene.
48 // Each CSector will actually be a platfrom specific class derived from CSector,
49 // like: CPs2Sector, CXboxSector or CNGC Sector
50 // The derived class (like, CPs2Sector), contains the platform specific data that
51 // identifies the sector in the engine. On the PS2, this might have a list of the
52 // meshes that make up the sector, maybe a list of the materials, probably a pointer to the
53 // collision data, and maybe a pointer to a structure containing sector flags.
54 // CSector provides functions like SetVisibility(), which call a platform specific
55 // function (PlatSetVisibility()), which calls the actual engine specific
56 // function, which performs whatever is necessary at a low level to make the
57 // sector visible/invisible (on the PS2, this might involve relinking a DMA chain,
58 // on other platforms, it might just involve flipping a bit.)
59 //
60 // The "engine programmers" have to implement and maintain the platform
61 // specific CPs2Sector (and CXBox.., etc) implementation.
62 //
63 // The "game programmers" simply use the CSector class, and will never need to know
64 // anything about the derived class, and their code will work across all three platforms.
65 //
66 // The End
