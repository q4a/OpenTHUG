23 /*
24  The base class for all screen elements. Screen elements include things like onscreen text, HUD sprites,
25  items in menus, etc. Every screen element has 2D coordinates and (rectangular) dimensions. Screen elements
26  can be "children" of other screen elements, whereby the child uses coordinates relative the upper left corner
27  of the parent. If the parent is moved, all its children will be moved along with it.
28 
29  A screen element also has a pair of scale values, which are independent of its base width and height. A
30  screen element will generally have a fixed base width and height, but the scale can be varied for animation
31  purposes, making the element appear to grow or shrink in size. Scale changes to an element are also passed to
32  children.
33 
34  The same idea applies to alpha and RGBA. RGBA is a static setting for the screen element, but alpha is a
35  separate morpheable multiplier that can be used for animation.
36 */
