11 // The CQuickAnim is a per-object interface to the CBonedAnimFrameData
12 // (which is raw animation data). Because multiple objects can be playing
13 // the same animation at the same time, we can't do anything fancy like
14 // caching pointers inside the CBonedAnimFrameData; however, since each
15 // object has its own CQuickAnim, we can store pointers or decompress data
16 // or whatever here. I've decided to give this class a p-line interface
17 // so that different platforms can implement different optimizations
18 // based on whether speed or memory is the bottleneck.
