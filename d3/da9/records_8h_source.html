18 // Notes:
19 // as a bit of an experiment, I'm writing the whole structure using
20 // dynamic containers,
21 // so each data type is more independent of the others
22 // but the constructor (and destructor) of the higher level types
23 // will have to create (and destroy) the dynamic instances of the others
24 //
25 // this gives us benefits, namely:
26 // No cross includes needed between header files if we split into seperate classes
27 // Number of elements in arrays (for high score tables) can be decided at run time.
28 //
29 // But has drawback:
30 // Uses more memory, as each element is allocated individually
31 // might be slower?
32 // more code? Dunno, that's why I'm experimenting....
