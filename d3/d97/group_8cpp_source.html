8  A Group is a set of textures, and a set of meshes or primitives that use those textures.
9 
10  Rendering is performed one group at a time.
11 
12  The textures for a group are uploaded, and then it is rendered.
13 
14  Whilst a group is being rendered, the uploading of the next group's textures
15  can start to be uploaded.
16 
17  This all happens asyncronously, whilst the CPU is doing the calculations for the next frame.
18 
19  The uploading/rendering cycle is syncronized by interrupts.
20 
21  The groups are rendered in this order:
22 
23  1) PrologueGroup - set up in render.cpp - flips the display buffers and initializes VU1 microcode
24  2) The groups linked by PrologueGroup->pNext, which represent groups of textures and polygons
25  3) pEpilogue, (the last in this list) - uploads fonts and sprites, allowing them to be rendered next
26 
27  after the groups are rendered, we can render the 2D elements, namely the sprites and fonts (see render.cpp)
