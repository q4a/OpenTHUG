79 // The purpose of this manager is just to update all the locked objects, by calling
80 // CMovingObject::LockToObject on all objects that have had Obj_LockToObject run on them.
81 // It is a manager so that it can be given a task with a priority such that it is called
82 // after all the objects are updated.
83 // The reason the lock logic cannot be done in the object update is because sometimes the parent's
84 // update might happen after the child's, causing a frame lag. This was making the gorilla appear
85 // behind the tram seat in the zoo when the tram was being skitched. (TT2324)
86 
87 class Manager : public Spt::Class
