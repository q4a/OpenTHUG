47  // byte padding, keeping the size of this class down. There are 4000 or so of these, so

50  // If a symbol is deleted and recreated by parse.cpp then this flag will get set.
51  // This is so that other code can use this to detect whether a symbol has changed due to a reload.
52  // For example, file.cpp checks the script mGotReloaded flags after loading a qb, and if reloaded
53  // it will then restart all existing CScript's that refer to it so that they do not crash due to
54  // invalid pointers.
55  // Similarly, if a qb gets unloaded any existing CScript's that refer to a script in it must be stopped.
56  // Also, in THPS, if the symbol NodeArray gets reloaded then name prefix information must be regenerated.
57  //

69  // This used to just store the upper checksum bits, because the lower bits are implied
70  // by the index of this entries list in the hash table.
71  // This was just to save space, because of the fairly large (8000ish) number of symbols.
72  // However, it makes the LookUpSymbol function a little bit slower because of the extra bit
73  // operations needed to compare, and also it makes the RemoveSymbol function more complex
74  // and slower because it has to search the whole symbol table in order to find the previous
75  // entry, so that the current one can be unlinked.
76  // In skate4, since we have the ability to unload qb's, this should reduce the requirement
77  // for such a large symbol table, so I decided to store the whole checksum instead.
78  // Simpler, faster and neater! Just uses a wee bit more space. Not a huge amount though,
79  // probably only 25K ish.
