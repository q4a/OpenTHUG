381 // returns the PROJECTED length of the next frame, for use in logic calculations
382 // done by averaging the frame lengths of the last N frames (default N = 4)
383 // at 60fps (NTSC), this would return 1/60
384 // at 50fps (PAL), this would return 1/50
385 // complications arise when the framerate changes
386 // we want the physics to move smoothly and consistently at all framerates
387 // however, all we know is the number of ticks the last frames took
388 // we also need to ensure that the total of all the "FrameLength" calls will
389 // reflect the actual time that has passed
390 // | | | | | | | | | | | | | | | | | |
391 // say we have 1 1 1 . 2 1 1 1 . 2 1 . 2 1 1 1 . 2
392 // (10x1 + 4x2 = 18 frames
393 //
394 //
395 // now, let's work in 1/60ths of a second for now....
396 // let's take the average of the last 4 frames, as the logic time we will return for this frame
397 //
398 // 1 1 1 1.25 1.25 1.25 1.25 1.25 1.25 1.5 1.5 1.25 1.25 1.25
399 //
400 // the reason we do the averaging is to smooth out discontinuities
401 // if we simply use the length of the previous frame, then if we were to be alternating between 1 and 2 frames
402 // then the logic would move 2 on frames of length 1, and move 1 on frames of length 2
403 // this would essentially make the skater visually move alternately .5x and 2x his actual speed
404 // this looks incredibly jerky, as there is a 4x difference per frame
405 // by smoothing it out, you would move 1.5 on every frame, giving relative motion of 0.6666 and 1.3333,
406 // this is still jerky, but half as much as the unsmoothed version.
