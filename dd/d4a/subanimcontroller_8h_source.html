19 // This file contains all the custom animation controllers
20 // which can be chained together by an animation channel
21 // to produce its final array of RT data for each frame.
22 // For example, an animated skeleton might need to:
23 // (1) grab interpolated keyframe data
24 // (2) flip the skeleton
25 // (3) rotate the skateboard by 180 degrees
26 // (4) perform IK on the leg bones to lock them to the skateboard
27 // (5) apply an additional keyframed anim to only the cloth bones
28 // (6) orient the head towards the closest pedestrian.
29 // In this case, there would be one animation channel with
30 // six animation controllers.
31 
32 // Eventually, we'll probably want to break up this
33 // code into a separate file for each animation
34 // controller, and perhaps create a factory class so that
35 // BlendChannel.cpp doesn't need to be updated every time
36 // a new animation controller is added.
